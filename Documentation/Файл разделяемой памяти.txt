Файл разделяёмой памяти создаётся демоном RadControl при запуске. Путь к нему
/tmp/radcontrol. Демон же поддерживает актуальность данных в этом файле.
Данные в файле хранятся в бинарном формате. Файл имеет следующую структуру:
1. Байт с битами состояния демона. Используются последние два бита.
Выставленный бит 0b00000001 означает, что демон работает, опущенный означает, что он был остановлен.
Выставленный бит 0b00000010 должен означать, что по каким-то причином демон не может писать показания в базу данных.
2. Байт с полным числом детекторов
3. Массив структур с данными для каждого детектора. (Здесь слово "структура" является термином языка С/С++) Эта структура - "struct DetectorData" - описана в файле DetectorData.h:

struct DetectorData{
  DetectorType type;//Тип - гамма, нейтронный или неизвестный
  DetectorState state;//Состояние (см.ниже)
  uint modbus_id;//modbus-адрес
  uint count;//счёт (последнее измерение)
  float doserate;//текущее показание мощности эквивалентной дозы, после преобразования
  uint16_t exposure,//экспозиция
       exposure_by_count;//экспозиция по счёту (для гамма здесь стоит 0)
  float sensitivity;//чувствительность (для гамма здесь стоит 0)
};

Тип детектора указывается в переменной "type" типа "enum class DetectorType". Тип может принимать значения:
*DetectorType::UNKNOWN_TYPE - неизвестный тип.
*DetectorType::GAMMA
*DetectorType::NEUTRON
Состояние детектора указано в переменной "state" типа "enum class DetectorState". Состояние может принимать значения:
*DetectorState::OK
*DetectorState::DISABLED - датчик отключён
*DetectorState::INIT - датчик работает, но не закончил первое измерение, и указанные пока значения не являются действительными
*DetectorState::NO_CONNECTION - связь с датчиком потеряна
*DetectorState::NO_CALIBRATION - датчик работает, но не удалось найти для него файл с калибровочными данными, поэтому указанный фон является недействительным
*DetectorState::UNKNOWN - случилось что-то непридвиденное.
Указанные в файле разделяемой памяти показания следует считать действительными, только если детектор находится в состоянии OK, в состоянии NO_CALIBRATION правильными являются все указанные параметры, кроме фона.

Соответственно, сдвиги в байтах для данных в файле разделяемой памяти таковы:
0	Байт состояния
1	Байт числа детекторов
2+i*sizeof(DetectorData)	Данные для i-го детектора

Заголовочный файл "DetectorData.h" содержит определения, необходимые для работы с файлом разделяемой памяти. Этот заголовочный файл включают все программы, взаимодействующие с файлом разделяемой памяти, а именно демон RadControl и device-сервер (RadCtrl).

Запись в файл разделяемой памяти осуществляет только RadControl. Читают из него RadControl и device-server.

Эти процессы отображают файл разделяемой памяти в своё виртуальное адресное пространство и далее работают с данными в нём, как с обычными переменными, лежащими в памяти. Для этого используется механизм mmap.
Документацию по нему можно почитать командой:
man mmap
